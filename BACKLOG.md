## Bug Log

Document bugs encountered during development, their root cause, and how they were resolved.

| # | Description | Root Cause | Resolution | Status |

| 1 | `-` character showed as `6` mid-animation | Expected: space→`-` is 43 spool steps × 60ms = 2.58s; screenshot taken at 2s hit index 33 = `6`. Fully settled at 4s. | Not a bug — raised screenshot wait from 1.5s → 4s to confirm. | Resolved |
| 2 | Demo not auto-starting on load | `startDemo()` was only called inside a click-to-start handler | Removed click gate; call `startDemo()` directly in `document.fonts.ready` | Resolved |
| 3 | Tiles showed beige/cream background instead of black with white letters | SplitFlapTV-Regular is a stencil/inverse font — glyph fills the tile face, letter is a cutout. | 7 | 2-panel animation regression — new char flashed before flip | Refactor pre-loaded new char into visible front panels before CSS sweep | Restored 4-panel architecture: back panels (`.tb`/`.bb`) hold new char at -90°/+90° (invisible); reveal on flip | Resolved |

| 28 | Board page fullscreen tiles smaller than homepage fullscreen | `board.js` called `document.documentElement.requestFullscreen()` making `<html>` the fullscreen element, so `body:fullscreen` CSS selectors never matched — board stayed at `max-width: 1200px` instead of `100vw`. | 29 | Phone couldn't reach server on LAN — QR scan had no response | Node.js on Windows binds to `::` (IPv6) by default; `::` does not accept IPv4 connections on Windows (unlike Linux). | 30 | `getLanIp()` could return virtual adapter IP (WSL/VMware) instead of WiFi IP | `os.networkInterfaces()` returns adapters in arbitrary order; first non-internal IPv4 might be `172.x.x.x` (Hyper-V/WSL) which is unreachable from phones. | Refactored `getLanIp()` to collect all candidates then prefer `192.168.x.x`, then `10.x.x.x`, over other ranges. | 31 | Phone disconnect returned board to QR pairing screen | `disconnected` case in `board.js` called `showQrScreen()`, hiding the board and resetting layout | Changed to call `displayRows(DISCONNECTED_ROWS)` — board stays open showing DISCONNECTED tiles; reconnecting phone resumes session without re-scanning QR | Resolved |
| 32 | Controller preview board too large on initial load — corrects itself after scrolling | `fitPreview()` called synchronously inside `document.fonts.ready.then()`; on iOS Safari, `grid.offsetWidth` is 0 at that point so the guard bails, leaving the preview unscaled (full native ~824px width). Scrolling triggers a resize/reflow that calls the resize listener and fixes it. | Wrapped `fitPreview()` in `requestAnimationFrame()` to defer until after layout paint; added `ResizeObserver` on `#preview-wrapper` as a secondary trigger for orientation changes. | Resolved |
| 33 | Controller preview characters doubled — same character appeared separately in top and bottom panel halves | Preview tiles use a simplified 2-panel structure (`tile-top`/`tile-bottom` directly containing `.tile-char`). Without flex centering on the panels, the block-level span renders at the top of each panel — result: two separate copies of the char. Also, `splitflap.css` default `translateY(0.75rem)` was sized for a 3rem tile (see bug #10 comment), not the 4.3rem preview tiles; correct offset is 4.3rem/4 = 1.075rem. | Added `display: flex; align-items: center; justify-content: center` to `#board-preview .tile-top/bottom` in `controller.css`. Overrode `translateY` to `±1.075rem` and `font-size` to `4.3rem` for preview tiles. | Resolved |
